# Bitovi Blog AI Agent

A local full-stack AI agent for answering questions grounded in articles from
the Bitovi blog.

The agent uses a retrieval-augmented generation layer to search an indexed copy
of the Bitovi blog before it answers. The backend crawls Bitovi blog articles,
chunks the extracted article text, embeds those chunks with a Hugging Face
sentence-transformer model, persists them in Chroma, and exposes a LangChain
ReAct agent backed by Claude Haiku through Anthropic. The frontend is a Next.js
single-page app that sends questions to the FastAPI service and renders the
agent's answer with cited source articles.

## How It Works

At runtime, the FastAPI backend receives a question and passes it to the
LangChain agent. The agent has access to a `search_blog` retrieval tool backed
by Chroma. For Bitovi-related questions, it can query the RAG layer, inspect the
retrieved article chunks, reformulate the search once if results are weak, and
then produce a concise answer with the source articles that informed it.

The RAG layer is built ahead of time by `ingest.py`. Ingestion loads Bitovi blog
pages, normalizes article text and metadata, splits the articles into chunks,
embeds those chunks, and stores them locally in `chroma_db/`.

## What It Uses

- FastAPI for the Python API
- LangChain for the ReAct agent, retrieval tool, and Anthropic integration
- Anthropic `claude-haiku-4-5` for answer generation
- Hugging Face `sentence-transformers/all-MiniLM-L6-v2` for embeddings
- `huggingface-cli` for Hugging Face model authentication and downloads
- Chroma for the local vector store
- Next.js, React, and TypeScript for the browser UI

## Repository Layout

```text
.
|-- chain.py              # Agent, retrieval tool, retriever, and source extraction
|-- config.py             # Model, embedding, Chroma, and blog settings
|-- ingest.py             # Blog crawler, cleaner, chunker, and Chroma ingester
|-- main.py               # FastAPI app and API models
|-- pyproject.toml        # Python project metadata and dependencies

|-- requirements.txt      # Python dependency export
|-- tests/                # Backend unit and integration tests
|-- eval/                 # Golden dataset and evaluation runner
`-- frontend/
    |-- app/              # Next.js app router UI and API route
    |-- package.json      # Frontend scripts and dependencies
    `-- public/           # Static frontend assets
```

Runtime-generated directories:

- `chroma_db/` stores the local Chroma index.
- `.cache/` stores cached crawled blog documents used to rebuild the index
  without re-scraping the site.
- `logs/` may contain local run logs.
- `frontend/.next/` is generated by Next.js.
- `frontend/node_modules/` is generated by npm.

## Prerequisites

- Python 3.11 or 3.12
- Node.js 20+
- `uv`
- npm
- An Anthropic API key in `ANTHROPIC_API_KEY`

The embedding model downloads from Hugging Face the first time it is used, so
the first ingest or API startup can take longer than later runs.

## Setup

Install the Python dependencies from the repository root:

```bash
uv sync --python 3.12
```

Log in to Hugging Face before using the embedding model:

```bash
uv run huggingface-cli login
```

Install the frontend dependencies:

```bash
cd frontend
npm install
cd ..
```

Create a root `.env` file:

```bash
ANTHROPIC_API_KEY=your_api_key_here
```

You can also export the variable in your shell instead:

```bash
export ANTHROPIC_API_KEY=your_api_key_here
```

## Build The Index

Build the RAG layer before asking questions:

```bash
uv run python ingest.py --reset
```

The first run scrapes the Bitovi blog and writes the normalized article
documents to `.cache/bitovi_blog_documents.json`. Later runs, including
`uv run --package bitovi-project python ingest.py --reset`, reuse that document
cache by default so resetting Chroma does not re-scrape the site.

When scraping is needed, the ingester tries `https://www.bitovi.com/sitemap.xml`
first and filters for blog URLs. If the sitemap load fails, it falls back to a
recursive crawl from `https://www.bitovi.com/blog`.

During ingestion, the RAG layer:

1. loads matching blog pages,
2. extracts article text and metadata,
3. splits article text into 500-token chunks with 50-token overlap,
4. embeds chunks in batches of 64,
5. persists the Chroma collection to `./chroma_db`.

Use `--reset` when you want to delete and rebuild the existing index. Rebuild
the index after changing the embedding model or chunking settings.

Use `--refresh-cache` when you want to re-scrape the blog and replace the
cached documents:

```bash
uv run python ingest.py --reset --refresh-cache
```

## Run Locally

Start the backend from the repository root:

```bash
uv run uvicorn main:app --reload
```

FastAPI runs on `http://localhost:8000`.

In another terminal, start the frontend:

```bash
cd frontend
npm run dev
```

Next.js runs on `http://localhost:3000`.

Open `http://localhost:3000`, enter a question, and the UI will call the AI
agent at `http://localhost:8000/ask` by default. The backend allows CORS
requests from `http://localhost:3000` and `http://127.0.0.1:3000`.

## Configuration

Backend settings live in `config.py`:

| Setting | Default | Purpose |
| --- | --- | --- |
| `LLM_MODEL` | `claude-haiku-4-5` | Anthropic model used for generation |
| `EMBED_MODEL` | `sentence-transformers/all-MiniLM-L6-v2` | Hugging Face embedding model |
| `BLOG_URL` | `https://www.bitovi.com/blog` | Recursive crawl fallback root |
| `CHROMA_DIR` | `chroma_db` | Local Chroma persistence directory |
| `COLLECTION_NAME` | `bitovi_blog` | Chroma collection name |
| `RETRIEVER_K` | `3` | Number of chunks the agent retrieves per search |

Frontend API configuration:

- `NEXT_PUBLIC_FASTAPI_URL` controls where the browser UI sends questions.
  Default: `http://localhost:8000`.
- `FASTAPI_URL` controls the optional Next.js server route at
  `POST /api/ask`. Default: `http://localhost:8000`.

The current UI calls FastAPI directly from the browser. The `/api/ask` route is
available if you want to call the backend through Next.js server-side routing.

## API

### Health Check

```http
GET /health
```

Response:

```json
{
  "status": "ok"
}
```

### Ask The Agent

```http
POST /ask
Content-Type: application/json

{
  "question": "What does the Bitovi blog say about React performance?"
}
```

Response:

```json
{
  "answer": "string",
  "sources": [
    {
      "title": "string",
      "url": "string"
    }
  ]
}
```

Errors:

- `422` when the question is blank or the request body is invalid.
- `503` when the Chroma index is missing. Run `uv run python ingest.py --reset`.
- `502` when the agent request fails, usually because the API key or RAG index
  is not available.

## Testing

Run the Python test suite from the repository root:

```bash
uv run pytest -q
```

The tests are isolated from live Anthropic, Hugging Face, Chroma, and network
calls by using fake executors, retrievers, vector stores, and FastAPI
monkeypatching.

Test coverage is organized by behavior:

- `tests/conftest.py` provides shared fake documents, retrievers, vector stores,
  and executors.
- `tests/test_retriever.py` covers source deduplication, agent step document
  extraction, and retriever configuration.
- `tests/test_agent.py` covers `answer_question()` behavior with mocked agent
  and retriever paths.
- `tests/test_api.py` covers `/health`, `/ask`, request validation, missing
  vector store handling, and unexpected agent failures.
- `tests/test_ingest.py` covers document cache reuse, forced cache refresh, and
  ingest argument propagation.
- `tests/test_eval.py` covers the local eval scoring helpers.

FastAPI may emit deprecation warnings for `@app.on_event("startup")`; those
warnings do not currently fail the suite.

## Evaluation

The evaluation harness lives in `eval/`:

- `eval/golden_dataset.json` contains curated questions and expected topic
  coverage.
- `eval/run_eval.py` runs the current agent against that dataset and prints JSON
  results with `average_score`, per-question `score`, `matched_topics`, and
  `missed_topics`.

Run it from the repository root:

```bash
uv run python eval/run_eval.py
```

The evaluator calls the real agent, so it requires:

- `ANTHROPIC_API_KEY` to be set.
- A populated `chroma_db/` index from `uv run python ingest.py --reset`.

The current scorer is a lightweight local heuristic. It checks normalized
keyword overlap against each expected topic and treats returned source URLs as
citation coverage. It is useful for quick regression checks, not a replacement
for deeper human review or a true LLM-as-judge evaluator.

## Troubleshooting

### `Vector store not found`

The backend could not find `./chroma_db`. Run:

```bash
uv run python ingest.py --reset
```

### Anthropic authentication errors

Confirm `ANTHROPIC_API_KEY` is available to the process running FastAPI:

```bash
echo "$ANTHROPIC_API_KEY"
```

If you are using `.env`, keep it in the repository root because `config.py`
loads environment variables from the backend working directory.

### First run is slow

The Hugging Face embedding model is downloaded and initialized on first use.
FastAPI also warms embeddings during startup.

### Browser cannot reach the API

Confirm the backend is running on `http://localhost:8000`. If you use a
different backend URL, set `NEXT_PUBLIC_FASTAPI_URL` before starting Next.js:

```bash
cd frontend
NEXT_PUBLIC_FASTAPI_URL=http://localhost:8000 npm run dev
```

### Ingestion finds no documents

The ingester depends on the public Bitovi sitemap or blog pages being reachable.
Retry later, or check whether `https://www.bitovi.com/sitemap.xml` and
`https://www.bitovi.com/blog` are accessible from your network.
